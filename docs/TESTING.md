# 测试体系

## 当前测试栈

- `Vitest`：执行单元测试和组件测试
- `Testing Library`：验证 React 组件行为
- `jsdom`：提供浏览器环境

## 当前覆盖重点

- 表单校验逻辑
- 登录服务的成功与失败路径
- 路由守卫在登录/未登录下的跳转行为

## 常用命令

```bash
npm run test
npm run test:watch
npm run test:coverage
```

## 编写原则

- 先测高价值行为，再补低层细节
- 优先覆盖容易回归的登录、路由、状态和错误提示
- 纯业务规则尽量写成可单测的函数
- 覆盖率报告用于辅助找盲区，不代替行为断言
