# 覆盖率说明

## 当前命令

```bash
npm run test:coverage
```

执行后会生成：

- 终端文本摘要
- `coverage/index.html` 可视化报告
- `coverage/coverage-summary.json` 供 CI 或后续脚本使用

## 当前定位

覆盖率目前用于建立基线和发现盲区，不作为硬性拦截阈值。

## 当前重点已覆盖

- 登录表单校验
- 登录成功与失败路径
- 受保护路由守卫
- 顶部布局的登出交互
- 全局消息提示的展示、关闭和自动消失

## 使用建议

- 优先提升登录、路由守卫、共享状态和错误处理的覆盖率
- 不要为了数字牺牲测试价值
- 当关键流程增加时，先补代表性测试再扩展阈值
- 下一批值得补的区域是首页路由编排、控制台页加载态，以及共享类型和只读展示组件
