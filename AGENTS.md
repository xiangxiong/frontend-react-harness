# AGENTS.md

这是给人和编码智能体看的简版导航文件。请先从这里开始，再沿着文档入口继续阅读，不要靠猜。

## 项目目标

构建一个适合智能体长期协作的 React 前端工程。相比一次性的长提示词，更优先依赖清晰边界、版本化上下文和可执行规则。

## 建议先读

1. 阅读 `README.md` 了解本地命令
2. 阅读 `ARCHITECTURE.md` 了解代码边界
3. 阅读 `docs/index.md` 了解文档地图
4. 阅读 `docs/exec-plans/active/` 下的当前执行计划

## 工作规则

- 保持 `app` 层足够薄，领域 UI 放在 `src/domains/*/ui`
- 数据整理和领域决策放进 `service` 或 `types`，不要堆在 React 入口文件里
- 反复重要的规则尽量编码进脚本或文档，而不是每次靠提示词重复
- 当行为、架构或约束变更时，同步更新文档

## 必跑检查

- `npm run lint`
- `npm run build`

## 交付方式

优先提交小而清晰的变更。说明本次改了什么、为什么改、以及如何验证。
